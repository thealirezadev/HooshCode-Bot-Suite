## 📖 معرفی

**HooshCode** یک اکوسیستم متشکل از ۶ ربات تلفیقی روی پیام‌رسان [بله](https://docs.bale.ai/) است که هرکدام یک نیاز مشخص کاربران فارسی‌زبان (به‌ویژه برنامه‌نویسان) را پاسخ می‌دهند؛ از تبدیل فایل و OCR گرفته تا ساخت QR Code، تولید پسورد امن، اخبار/قیمت لحظه‌ای بازار و چت با هوش مصنوعی.

نکته‌ی متمایزکننده‌ی این پروژه نسبت به مجموعه‌ربات‌های معمول، **یکپارچگی معماری** است:

- 🗄️ همه‌ی ربات‌ها به یک **دیتابیس SQLite مشترک** وصل‌اند (نه ۵ فایل دیتابیس پراکنده)
- 🛠️ یک **پنل مدیریت متمرکز** در ربات اصلی، مدیریت محتوا/کاربران/تیکت‌ها/نظرسنجی همه‌ی ربات‌ها را در یک‌جا ممکن می‌کند
- ♻️ منطق مشترک (بررسی عضویت کانال، نظرسنجی، تیکت پشتیبانی، کلاینت API بله) در ماژول‌های قابل‌استفاده‌ی مجدد (`utils/`) قرار دارد تا کد در ۶ ربات تکرار نشود

| 📄 **هوش کد \| تبدیل** | تبدیل فایل | عکس↔PDF، **OCR فارسی/انگلیسی محلی**، رمزگذاری اختیاری PDF خروجی |
| 🔐 **هوش کد \| رمز** | تولید پسورد | پسورد تصادفی امن، PIN، پسورد WiFi/خوانا، ذخیره‌ی پسوردهای شخصی |
| 📱 **هوش کد \| QR** | ساخت QR Code | ۳ استایل + ۸ رنگ، QR وای‌فای/کارت‌ویزیت، سیستم دعوت دوستان |

## ✨ قابلیت‌های مشترک بین ربات‌ها

تمام ربات‌ها (به‌جز ربات اخبار که در کانال فعالیت می‌کند) این سه قابلیت مشترک را دارند:

### ⭐ نظرسنجی پس از هر خدمت
بعد از هر عملیات (ارسال سورس کد، تبدیل PDF، ساخت QR، تولید پسورد، پاسخ چت) سه دکمه‌ی «👍 راضی بودم / 😐 معمولی بود / 👎 راضی نبودم» نمایش داده می‌شود. ادمین در پنل مرکزی، **گزارش خلاصه** (تعداد و درصد رضایت) به تفکیک هر ربات را می‌بیند.

### 🆘 پشتیبانی داخل ربات
کاربر با زدن دکمه‌ی «ارتباط با پشتیبانی»، پیامش مستقیماً به‌صورت **تیکت** در دیتابیس مشترک ثبت و به ادمین اطلاع‌رسانی می‌شود. ادمین از همان پنل مرکزی پاسخ می‌دهد و ربات به‌صورت خودکار پاسخ را **از همان رباتی که کاربر در آن پیام داده بود** برایش ارسال می‌کند.

### 🔒 عضویت اجباری در کانال
بررسی یکپارچه‌ی عضویت کاربر در کانال‌های تعیین‌شده، با کیبورد آماده‌ی «عضویت + بررسی مجدد».

### چرا این معماری؟

| ویژگی | توضیح |
|---|---|
| **دیتابیس مشترک** | اطلاعات کاربر یک‌بار ثبت می‌شود و در هر ۵ ربات در دسترس است؛ آماری مثل «این کاربر چند ربات را امتحان کرده» به‌سادگی قابل استخراج است. |
| **پنل ادمین متمرکز** | دیگر نیازی به `/admin` جدا در هر ربات نیست؛ همه‌چیز از یک‌جا (ربات اصلی) مدیریت می‌شود. |
| **کلاینت API مشترک** (`utils/bale_client.py`) | فراخوانی‌های تکراری به API بله (ارسال پیام، عکس، فایل، بررسی عضویت) یک‌بار نوشته و در ۶ ربات استفاده می‌شود. |
| **پراسس‌های مجزا** (`run_all.py`) | هر ربات در پراسس خودش اجرا می‌شود؛ کرش یک ربات، بقیه را قطع نمی‌کند. |

## 🐛 رفع باگ‌های کلیدی نسخه‌ی قبلی

در بازبینی کد اولیه، این مشکلات شناسایی و رفع شدند:

1. **گیر کردن در پنل ادمین** — در نسخه‌ی قبلی، پس از ورود به پنل با `/admin`، هیچ دکمه‌ی صریحی برای خروج وجود نداشت. دکمه‌ی «بازگشت» داخل پنل فقط منوی ادمین را دوباره نمایش می‌داد و `admin_sessions` کاربر هرگز پاک نمی‌شد؛ در نتیجه پیام‌های بعدی کاربر (مثلاً دکمه‌های منوی عادی) بلااستفاده در شاخه‌ی پردازش ادمین گم می‌شدند. **راه‌حل:** دکمه‌ی «🚪 خروج از پنل ادمین» اضافه شد که session را به‌صراحت پاک می‌کند؛ علاوه‌بر این، در مسیر `back_to_main` نیز یک خروج دفاعی اضافه شده است.
2. **تداخل بررسی عضویت با هندلینگ کالبک‌های ادمین** — اکنون callback های ادمین قبل از بررسی عضویت کاربر هندل می‌شوند تا ادمین هیچ‌وقت پشت دیوار «عضویت در کانال» گیر نکند.
3. **دیتابیس‌های پراکنده‌ی JSON/SQLite در هر ربات** — جایگزین شد با یک دیتابیس واحد و یک API مرکزی (`database/db.py`).

## 🧠 ربات چت (قابلیت هوش مصنوعی)

- مدل پیش‌فرض برای مکالمه‌ی روزمره: `meta-llama/llama-3.3-70b-instruct:free`
- اگر پیام کاربر شبیه درخواست کدنویسی باشد (کلیدواژه‌هایی مثل «کد بنویس»، نام زبان‌های برنامه‌نویسی، عبارات خطا)، به‌صورت **خودکار و موقت** به مدل `qwen/qwen3-coder:free` سوئیچ می‌کند.
- کاربر همچنین می‌تواند از منوی «🔄 تغییر مدل» به‌صورت دستی مدل دلخواهش (از میان چند مدل رایگان OpenRouter) را انتخاب کند.
- **حافظه‌ی مکالمه** در دیتابیس مشترک ذخیره می‌شود (نه فقط در RAM)، یعنی با ری‌استارت سرور هم از بین نمی‌رود. دستور `/clear` یا دکمه‌ی «پاک کردن حافظه» برای شروع گفتگوی تازه.
- ارتباط با OpenRouter از طریق SDK رسمی `openai`، بدون قرار گرفتن هیچ کلید API در کد (همه از `.env`).

## 🔍 ربات تبدیل (OCR و رمزگذاری PDF)

- **OCR محلی و رایگان** با [`easyocr`](https://github.com/JaidedAI/EasyOCR) که از فارسی و انگلیسی به‌خوبی پشتیبانی می‌کند و نیازی به سرویس ابری (و کلید API) ندارد.
- هنگام ساخت PDF از عکس‌ها، از کاربر پرسیده می‌شود که آیا می‌خواهد روی فایل خروجی رمز بگذارد؛ با ارسال رمز، PDF با `pypdf` رمزگذاری می‌شود یا با دستور `/skip` این مرحله رد می‌شود.

## 📈 ربات اخبار (نمودار هفتگی)

علاوه‌بر گزارش روزانه‌ی بازار (طلا/ارز/کریپتو) که هر روز ساعت مشخص با سنجاق خودکار در کانال منتشر می‌شود، این نسخه هر ۷ روز یک‌بار یک **نمودار روند قیمت هفته‌ی اخیر** (دلار، طلا، بیت‌کوین) با `matplotlib` رسم و در کانال ارسال می‌کند.

## ⚙️ نصب و اجرا

### پیش‌نیازها
- Python 3.10 یا بالاتر
- توکن ربات از [پنل توسعه‌دهندگان بله](https://docs.bale.ai/)
- کلید API از [OpenRouter](https://openrouter.ai/) (برای ربات چت)

### ۱. کلون پروژه و نصب وابستگی‌ها

```bash
git clone https://github.com/yourusername/hooshcode-bot-suite.git
cd hooshcode-bot-suite
python -m venv venv
source venv/bin/activate   # ویندوز: venv\Scripts\activate
pip install -r requirements.txt
```

### ۲. تنظیم متغیرهای محیطی

```bash
cp .env.example .env
```

سپس فایل `.env` را باز کرده و مقادیر واقعی (توکن‌ها، آیدی ادمین، کلید OpenRouter و ...) را وارد کنید. **این فایل هرگز نباید commit شود** (در `.gitignore` قرار دارد).

### ۳. اجرای همه‌ی ربات‌ها

```bash
python run_all.py
```

یا اجرای فقط چند ربات خاص:

```bash
python run_all.py main convert chat
```

### ۴. اجرای جداگانه‌ی یک ربات (مثلاً برای دیباگ)

```bash
python -m bots.main_bot.bot
python -m bots.chat_bot.bot
```

## 🔐 امنیت

- تمام توکن‌ها، کلیدهای API و آیدی‌های حساس **فقط** از فایل `.env` خوانده می‌شوند و هیچ‌جا در کد hardcode نشده‌اند.
- `.gitignore` به‌طور صریح فایل‌های `.env`، دیتابیس‌ها و داده‌های اجرایی را از ورود به ریپازیتوری مسدود می‌کند.
- رمزگذاری PDF با کتابخانه‌ی استاندارد `pypdf` انجام می‌شود؛ پسورد کاربر در سرور ذخیره نمی‌شود.
